# ヘルプコンテンツ一覧

各ページの `?` ボタンで表示されるヘルプモーダルの内容定義。
コード内の HelpButton children と1対1で対応する。

## デザイン方針

参考: Slack（検索+カテゴリブロック）、Loom（ミニマル3項目）、Spotify（折りたたみ+パーソナライズ）

- 構成: 「できること」箇条書き → 操作説明（短く） → 💡ヒント
- フォント: 本文 0.85rem、見出し 0.9rem + 下線
- 見出しは `<h3>` で統一、`<h2>` は使わない
- 💡ヒントは `.help-tip` ボックスで視覚的に区別
- 1モーダルあたり最大3セクション（スクロールさせない）
- 「ユーザーストーリー」という見出しは使わない（開発者用語）

## ページヘルプ（タイトル横 `?`）

### 監視対象サイト一覧 (Sites.tsx)

```
できること
• 登録済みサイトの監視状態を一覧で確認
• サイト名・URLで検索、ステータスや顧客でフィルタリング
• 新しい監視サイトの登録

クロールの実行
各サイトの「今すぐクロール」ボタンで最新データを取得します。
完了すると結果が自動反映されます。

結果の確認
「最終クロール」列の日時をクリックすると、取得データの詳細
（スクリーンショット・抽出データ・違反情報）を確認できます。

💡 ステータスの意味はステータスフィルター横の ? ボタンで確認できます。
```

### サイト管理 (SiteManagement.tsx)

```
できること
• 顧客→サイトの階層構造でサイトを管理
• 顧客名クリックでサイト一覧を展開
• サイト展開で詳細タブにアクセス

詳細タブ
• 契約条件 — 契約内容の確認・編集
• スクリーンショット — ベースライン画像の管理
• 検証・比較 — 検証実行と結果確認
• アラート — 違反アラートの確認

💡 ベースラインは1サイト1枚です。再キャプチャで上書きされます。
```

### 顧客マスター (Customers.tsx)

```
できること
• 顧客情報の登録・編集・削除
• 顧客名・会社名・メールで検索
• 有効/無効のステータスフィルタリング

操作方法
「+ 新規顧客登録」で追加、一覧の「編集」「削除」で変更できます。

💡 顧客を無効にしても関連サイトの監視は継続されます。
   無効化は管理状態の変更であり、監視停止ではありません。
```

### 契約条件管理 (Contracts.tsx)

```
できること
• サイトごとの契約条件（価格・決済方法・手数料・サブスク条件）を登録
• サイトフィルターで特定サイトの契約を絞り込み
• 契約のバージョン履歴を確認

設定項目
• 価格 — 通貨ごとの基準価格
• 決済方法 — 許可/必須の決済方法
• 手数料 — パーセンテージ・固定手数料
• サブスク条件 — 契約期間・解約ポリシー

💡 「現在」バッジの契約が監視基準です。
   違反検出はこの条件に基づいて行われます。
```

### アラート一覧 (Alerts.tsx)

```
できること
• 検出された違反・偽サイトアラートを重要度別に確認
• 重要度（緊急/高/中/低）と種別（契約違反/偽サイト）でフィルタリング
• 解決済みアラートの確認（半透明表示）

カードの見方
左ボーダー色が重要度を示します: 赤=緊急・高、黄=中、青=低。
偽サイトアラートにはTakeDown対応バナーが表示されます。

重要度の目安
• 緊急 — 即時対応（偽サイト確定、価格ゼロ等）
• 高 — 早急な確認（価格不一致、必須決済方法欠落等）
• 中 — 次回確認時に対応（手数料不一致、解約ポリシーずれ等）
• 低 — 必要に応じて確認（軽微な表示差異等）
```

### 偽サイト検知 (FakeSites.tsx)

```
できること
• 検知された偽サイトの一覧確認
• 偽ドメインと正規ドメインの類似度を比較
• 対応状況（未解決/解決済み）の追跡

スコアの見方
• ドメイン類似度 — ドメイン名の文字列の類似度（0〜1）
• コンテンツ類似度 — ページ内容の類似度（0〜1）
1に近いほど類似度が高く、偽サイトの可能性が高いことを示します。

💡 データは30秒ごとに自動更新されます。
```

### クロール結果レビュー (CrawlResultReview.tsx)

```
できること
• スクリーンショットと抽出データの並列確認
• 抽出データの手動編集・承認・却下
• HTML解析とOCR解析の精度比較

操作方法
• 左側のスクリーンショットはズーム・パン操作可能
• 右側のフィールドをクリックでスクリーンショット上をハイライト
• ✎ ボタンでフィールド値を直接編集

💡 信頼度スコアが低いフィールドから優先的に確認すると効率的です。
```

## インラインヘルプ（フィルター横 `?`）

### コンプライアンスステータス (Sites.tsx フィルター横)

```
準拠 (Compliant)
最新のクロール・検証で、契約条件との違反が検出されなかった状態です。
価格、決済方法、手数料、サブスクリプション条件の全てが契約通りに表示されています。

違反 (Violation)
契約条件との不一致が検出された状態です。
価格の相違、許可外の決済方法の表示、手数料の不一致、
サブスクリプション条件の違反などが含まれます。

保留 (Pending)
まだクロール・検証が実行されていない状態です。
サイト登録直後のデフォルトステータスで、
「今すぐクロール」ボタンで検証を開始できます。

エラー (Error)
クロールまたは検証処理自体が失敗した状態です。
サイトへの接続不可、タイムアウト、サーバーエラーなどが原因として考えられます。
時間をおいて再度クロールを実行してください。
```

## ツールチップ（Badge title 属性）

| コンポーネント | ステータス | ツールチップ |
|---|---|---|
| SiteRow Badge | compliant | 契約条件に準拠しています |
| SiteRow Badge | violation | 契約条件の違反が検出されました |
| SiteRow Badge | pending | クロール未実行または検証待ちです |
| SiteRow Badge | error | クロールまたは検証でエラーが発生しました |
| CrawlResultReview | pending | 抽出データの確認・承認待ちです |
| CrawlResultReview | approved | 抽出データが承認されました |
| CrawlResultReview | rejected | 抽出データが却下されました |
| CrawlResultReview | failed | データ抽出に失敗しました |
| AlertTab | resolved | このアラートは対応済みです |
| AlertTab | unresolved | このアラートは未対応です |
| ScreenshotTab | pending | 抽出データの確認待ちです |
| ScreenshotTab | confirmed | 抽出データが確認されました |
| ScreenshotTab | rejected | 抽出データが却下されました |
