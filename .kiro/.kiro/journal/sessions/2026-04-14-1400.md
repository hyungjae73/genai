## Summary
白画面問題のデバッグ。根本原因は CategoryManager.tsx が TypeScript interface を `type` キーワードなしで import していたため、Vite の ESM トランスパイル後に `Category` という名前の export が存在せずランタイムエラーが発生していた。`import type` に修正して解決。あわせて Dockerfile の CMD に `--force` フラグを追加し、起動時に Vite の依存関係キャッシュを強制再最適化するよう設定。

## Tasks Completed
- `CategoryManager.tsx` の import を `import type { Category, CategoryCreate }` に修正（HMR で即時反映確認）
- `frontend/Dockerfile` の CMD を `vite --host 0.0.0.0 --force` に変更
- フロントエンドイメージを再ビルド（`docker compose build frontend`）

## Decisions Made
- **`import type` を使う**: TypeScript interface/type は JS にトランスパイルされると消えるため、Vite の ESM モードでは `import type` を使わないとランタイムエラーになる。今後の型 import は全て `import type` を使う方針
