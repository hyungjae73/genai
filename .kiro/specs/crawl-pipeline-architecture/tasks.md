# Implementation Plan: Crawl Pipeline Architecture

## Overview

モノリシックな `_crawl_and_validate_site_async()` を4ステージ構成の `CrawlPipeline` に分解し、プラグインアーキテクチャで拡張可能にする。DB モデル拡張 → コアフレームワーク → プラグイン実装 → スケーラビリティ基盤 → API → フロントエンドの順で段階的に実装する。
